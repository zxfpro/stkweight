我希望帮我编写一个任务看板管理系统, 该系统包括5大部分内容


1 任务看板管理
2 番茄钟项目
3 基于AI的智能分拣任务和调度系统
4 对外暴露的mcp skills agent 接口
5 用户系统, 注册,登陆模块

## 任务看板管理
常见的看板任务管理系统,主要分为6个板块 预备池,就绪池, 执行池, 完成池, 阻塞池, 酱油池
任务可以被创建, 按照一定的逻辑在不同池子之间互相调度, 完成的任务卡片可以归档, 归档以后的任务卡片可以被管理, 删除, 和恢复
### 创建任务
任务主要分为, 标题, 具体内容, 负责人, 预估时间, 执行条件, tag 
1 只有对应的负责人才能将卡片移动到执行池
2 只有满足执行条件的才能将卡片移动到就绪池, 否则就在预备池中
3 预估时间匹配番茄钟的倒计时时间
4 tag 用来区分任务的大致类型
5 提供使用AI功能进行创建任务, 支持将模糊的大块的任务进行智能分割,切分成为1-2小时工作量的小任务,并合理分类

### 执行任务
1 同时,一个负责人只能执行n(默认是2,可以调整) 个任务, 同时执行, 番茄钟倒计时彼此独立控制
2 当负责人将卡片移动到执行池时, 默认开始执行,并在卡片上显示倒计时和番茄钟控制, 允许超时
3 当负责人将任务点击完成/移动到完成池时, 将任务卡片实际完成时间进行记录, 为AI预测时间估算提供数据支持
4 被阻塞(需要特定的条件) 将执行池移动到阻塞池中, 并实时监控条件是否满足, 如果满足, 将其调回执行池, 长期阻塞的移动会就绪池,并更新事件状态

### 完成任务
1 当任务被移动到完成池, 任务完成, 
2 完成的任务, 可以被归档, 记录相关信息和日期

### 任务调度
1 用户可以将任务卡片做调度
2 允许AI做自动调度任务
3 允许外部大模型通过MCP/skill等能力调度任务管理系统

## 番茄钟
番茄钟能力可以帮助人类进行注意力的集中, 帮助人们更好的完成任务
番茄钟分为两个时间, 一个是任务预计总时间, 另一个是单一番茄的时间
### 启动番茄钟
当任务卡片被移动到执行池时, 番茄钟被启动, 任务预计总时间读取任务被创建时的预计时间
开始倒计时(默认时间时25分钟, 休息5分钟) 最终任务完成时结算中消耗时间

### 番茄钟控制
每一个卡片都可以独立计时一个番茄钟, 每个番茄钟, 都有暂停, 继续, 休息, 跳过休息,完成等功能来丰富番茄钟的操作
任务完成后, 自动计算任务耗时, (不算休息时间) 最终人类可以查看和编辑时间
这个时间会为AI 预估时间作为重要参考


## 基于AI的智能分拣任务和调度系统

基于AI 的智能分拣系统, 用户可以输入一个模糊的大任务, AI根据之前的经验, 合理的切分任务(符合因故关系逻辑) 使其达到合理的工作量和工作时间,并提供一个表格让用户查看和编辑, 确认最终的任务, 做好合理的标签和任务归属

2 用户可以委托AI 来调度任务,创建完任务以后, AI会通过多种渠道(包括与用户沟通, 检测或者查看网络等手段) 来实时监控扫描任务的前置条件, 一开始所有任务都会放到预备池, 一旦某一任务的前置条件满足, 就将任务移动到就绪池, 当用户完成其他任务, (执行任务有空缺时, 工作时段) 主动推送就绪的任务到执行池并提示用户, (当然用户可以拒绝, 就不推送, 或者拉取任务) 推送任务要考虑到与之前任务的相关性, 考虑到任务的优先级等等, 最终执行完成后自动调度到完成池,归档,阻塞池等等

3 AI会不断学习数据库中的任务划分, 任务打标签, 和任务的时间预估, 并建立机器学习模式, 大模型提示词优化等潜在的模块, 来不断的个性化的预测和划分



## 对外暴露的mcp skills agent 接口
程序的主要功能都要对外提供mcp 服务, 可以支持Claude Code 进行链接, 实现针对性的skills 模块和对应的 claudecode 的 plugins 
目标是可以让第三方模型来控制该项目

## 用户系统
项目需要提供用户模块, 用户可以用邮箱注册, 登陆, 用谷歌账号快捷登录, 不同的数据要做用户隔离, 用户可以创建多个AI角色, 每个角色就是一个负责人, 可以对外链接和使用Claude Code 进行重型任务的工作
